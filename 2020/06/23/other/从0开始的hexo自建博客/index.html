图片插入问题

在hexo中使用本地图片的语法和MD语法不一致，导致本地编辑器中正常的图片在部署后就不正常了，

官方给出的三种方法都无法满足：https://hexo.io/zh-cn/docs/asset-folders

  1. 放到 source/images 文件夹，使用 ![](/images/image.jpg) 访问。能解决网页上的显示问题，但会导致本地编辑器不能正常显示。

  2. 将 config.yml 文件中的 post_asset_folder 选项设为 true 来打开。

    打开该功能后，Hexo 会在你每次通过 hexo new 命令创建文章时自动创建一个同名的文件夹，将图片放到该文件夹，使用相对路径访问。但在网页首页的图片还是不能正确显示。因为生成静态代码时会自动在public下创建一个文章名称的文件夹，将MD文件和对应的图片都放在了同一个目录下。。。。

  3. 使用标签插件，本地编辑器无法显示。

解决方法

使用 CodeFalling/hexo-asset-image 插件。

  1. 首先确保已经将 config.yml 文件中的 post_asset_folder 选项设为 true。

托管到github pages

创建github仓库

新建一个repository ，命名为 <你的 GitHub 用户名>.github.io，这样创建的pages访问地址就为<你的 GitHub 用户名>.github.io。如果以xxx.github.io为repository名，那么访问地址就为<你的 GitHub 用户名>.github.io/xxx.github.io。

如果不是User Pages还要在Setting-GitHub Pages里面设置资源的分支。

部署

安装 hexo-deployer-git

修改 _config.yml 配置，把刚刚复制的仓库地址粘贴到 repo: 后边
